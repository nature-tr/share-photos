└─────────────────────────┘              │

┌─────────────────────────┐              │     ┌───────────────────────────┐
│   微信小程序             │              ├────▶│      Fastify Server       │
│  (Taro 4 + React)       │  ── HTTPS ──┤     │   (Node.js + TypeScript)  │
└─────────────────────────┘              │     │ ┌───────────────────────┐ │
                                         │     │ │  Routes / Controllers │ │
                                         │     │ │  Services             │ │
                                         │     │ │  Repositories         │ │
                                         │     │ │  Storage Driver       │ │
                                         │     │ │  Cron (cleanup)       │ │
                                         │     │ └───────────────────────┘ │
                                         │     └───────────────────────────┘
                                         │               │             │
                                         │               ▼             ▼
                                         │    ┌───────────────┐  ┌──────────────┐
                                         └────│  SQLite (.db) │  │ FileSystem   │
                                              │  data/app.db  │  │  storage/... │

- 前后端同机部署，Fastify 提供 API + 文件流式返回。
- Nginx 反向代理 + Let's Encrypt SSL。
- PM2 进程守护 + 零停机重载。

## 2. 后端模块划分

```
backend/src/
├── app.ts                    # Fastify 实例装配（插件、路由、错误处理）
├── server.ts                 # 启动入口
├── config/                   # 环境变量、运行配置
├── db/
│   ├── client.ts             # better-sqlite3 + drizzle 实例
│   ├── schema.ts             # Drizzle 表定义（users/shares/photos/contributors/refreshTokens）
│   └── migrations/           # SQL 迁移文件
├── modules/
│   ├── auth/                 # 注册 / 登录 / refresh / logout / me
│   ├── share/                # 创建/列表/详情/重命名/续期/结束/硬删除 + 贡献者管理
│   ├── photo/                # 上传/删除/批量删除/缩略图/原图/zip
│   ├── viewer/               # 凭码访问（无需登录）+ 申请加入
│   └── cleanup/              # 过期分享清理 cron
├── infra/
│   ├── storage/              # 本地存储：ensureDirs / removeDirs / safeUnlink
│   ├── image/                # sharp：缩略图/中等图 JPEG + WebP
│   └── archive/              # archiver 流式 zip
├── plugins/                  # auth 装饰器 / CORS / rate-limit / error-handler
└── common/                   # errors / id / time
```

### 模块职责一览

| 模块 | 职责 | 关键依赖 |
| --- | --- | --- |
| `auth` | 注册/登录/refresh/登出/当前用户 | `@fastify/jwt`, `argon2` |
| `share` | 分享生命周期：CRUD/续期/结束/删除/重命名 | `photo`, `infra/storage` |
| `photo` | 上传、sharp 处理、删除、批量操作 | `infra/image`, `infra/storage` |
| `viewer` | 凭码查看、申请加入 | `share`, `photo` |
| `cleanup` | 每分钟扫描过期分享，删除文件+DB | `share`, `photo` |
| `infra/storage` | 本地存储抽象 | — |
| `infra/image` | sharp 封装 | `sharp` |

## 4. 核心流程

### 4.1 登录持久化

**Web**：Cookie `httpOnly` + `Secure` 存储 refresh token，内存存 access token。

**小程序**：
- 登录时写入 `Taro.setStorageSync` 存 user + accessToken + refreshToken
- 启动时 `getUserFromStorage()` 同步恢复（绕过 zustand 订阅时序问题）
- `checkAuth()` 后台验证 / 自动刷新 / 失败清理

### 4.2 图片处理

- 上传时 sharp 同步生成：缩略图 JPEG/WebP + 中等图 JPEG/WebP
- 前端请求缩略图时根据 `Accept` 头协商返回 WebP 或 JPEG
- 像素上限：268MP（手动 metadata 检查，sharp 0.33 无链式 API）

### 4.3 全局任务进度

- zustand `task.store` 追踪所有活跃上传/下载
- `GlobalProgress` 组件注册在 `App.tsx`，任何页面可见浮窗进度条
- 切换页面不丢失状态（请求本身继续执行）
- 完成 3 秒后自动消失

## 7. 横切关注点

- **鉴权**：Fastify `authenticate` 装饰器，受保护路由统一 `preHandler`
- **限流**：`@fastify/rate-limit`，登录 10/min，凭码 30/min，其他 60/min
- **错误处理**：`BizError` 统一包装，Zod 校验 → 400，未捕获 → 500
