# 在线共享相册 — 移动端适配

> 因为查看者主要是移动端用户，整个 Viewer 流程是移动端体验的重中之重。
> 本文档说明设计思路、技术现实与实现细节。

---

## 1. Web 不能直接做到的事（必须先对齐预期）

**浏览器没有 API 可以直接把图片写入手机相册（Photos / 相册）。**

可做与不可做：

| 行为 | iOS Safari | Android Chrome | 备注 |
| --- | --- | --- | --- |
| `navigator.share({ files })` 唤起系统面板 | ✅ 16.4+ | ✅ | **最接近"进相册"** ：用户可点"存储图像"/"保存到相册" |
| `<a href download>` 触发下载 | 进文件 App | 进 Downloads | **不直接进相册** |
| 用户长按 `<img>` → "添加到照片" | ✅ 系统级 | ⚠️ 部分支持 | 一张张操作 |
| zip 打包下载 | 文件 App 解压后再操作 | Downloads 解压后 | **手机上很难用** |

因此移动端的最佳路径是 **Web Share API**。

---

## 2. 实现策略

### 2.1 设备 / 能力检测

`frontend/src/composables/useDevice.ts` 提供：

- `useDevice()`：响应式 `isMobile`（`pointer:coarse` 或宽度 ≤ 768）
- `canShareFiles()`：当前设备是否支持 `navigator.share({ files })`

### 2.2 单张保存

`frontend/src/utils/download.ts` 的 `saveImage(url, opts)`：

```
preferShare && canShareFiles  →  navigator.share({ files: [file] })
                              ↓ 用户取消
                              return 'cancelled'
                              ↓ 不支持
                              fallback：触发浏览器下载
```

返回 `'shared' | 'downloaded' | 'cancelled'`。

### 2.3 批量保存（核心改进）

Web Share API **支持一次性 share 多个文件**：iOS 16.4+ Safari 与 Android Chrome 都允许 `navigator.share({ files: [...] })`，用户在系统面板**点一次「存储图像」**即可把整批图片**全部保存到相册**。

`utils/download.ts` 提供三种批量方式：

| API | 用途 | 实现关键 |
| --- | --- | --- |
| `saveImagesViaShare()` | 一次性 share 全部 → 用户点一次进相册 | 自动按 200MB 分批；超大相册分多次弹面板 |
| `saveImagesSequentially()` | 逐张 share（兜底） | 兼容老浏览器 |
| `saveImagesToDirectory()` | 桌面端选文件夹批量写入 | File System Access API（Chrome/Edge） |

**分批阈值**：单次 share 累计大小 ≤ 200MB（保守值）。超过会自动拆批，每批弹一次面板。

### 2.4 Viewer 页面下载入口

点击顶部「下载」按钮 → 弹出 Dialog，根据当前浏览器能力 + 设备类型动态展示选项：

| 选项 | 触发条件 | 标记 |
| --- | --- | --- |
| **一次性保存到相册** | `canShareFiles()` 为 true（移动端为主） | 推荐 ⭐ |
| **保存到本地文件夹** | 桌面 + Chrome/Edge（File System Access API） | 推荐 ⭐ |
| **打包 zip 下载** | 始终显示 | — |
| **逐张保存到相册 / 下载** | 始终显示（兜底） | — |

进度条显示：`准备图片(3/12)`、`等待系统面板`、`第 1/2 批` 等清晰阶段反馈，可随时取消。

### 2.5 大图查看器（PhotoSwipe）

- 默认手势：双指缩放、左右滑切换、单击隐藏 UI、垂直拖拽关闭
- 自定义按钮 **"保存"**（toolbar 第 8 位）：
  - 移动端 → Web Share API（用户系统面板选"存储图像"）
  - 桌面 → 直接下载

## 3. 响应式断点

```
默认（桌面）：> 768px
平板/手机   ：≤ 768px      → grid 改为 3 列、按钮改图标、顶栏折叠
超小屏       ：≤ 380px      → grid 改为 2 列（iPhone SE 等）
```

每个页面的具体调整：

- **App.vue 顶栏**：移动端 logo 文字隐藏、用户名隐藏、导航变图标按钮
- **HomePage**：标题缩小，输入 + 按钮改竖排，feature 卡片 2 列
- **LoginPage / RegisterPage**：卡片紧凑化，顶端对齐
- **MySharesPage**：卡片单列，操作按钮自动占满
- **NewSharePage**：表单改竖排、缩略图 3 列、底部操作栏改竖向、底部留白避免被遮挡
- **ViewerPage**：网格 3 列（小屏 2 列）、顶栏精简、保存按钮常驻

## 4. 触屏与无 hover 处理

```css
@media (pointer: coarse) {
  .quick-save { opacity: 1; }     /* 缩略图右下角下载按钮常驻 */
  button, .t-button { min-height: 40px; }  /* 触摸目标 ≥ 40px */
}
```

全局：
- `body` 设置 `-webkit-tap-highlight-color: transparent`，避免点击有蓝色高亮
- `viewport-fit=cover`，配合未来用 `env(safe-area-inset-*)` 适配 iOS 全面屏

## 5. 用户引导

- 移动端首次点"逐张保存到相册"前，弹 Notify 说明操作流程
- Dialog 里每个选项卡片自带说明文字
- 大图查看器右下角"保存"按钮 hover/title 文案根据 share 支持自动切换

## 6. 兼容性矩阵

| 设备 | 浏览器 | Web Share | 单张保存 | 批量保存 |
| --- | --- | --- | --- | --- |
| iPhone（iOS 16.4+） | Safari | ✅ | 系统面板 → 存储图像 → 进相册 | 逐张 |
| iPhone（iOS < 16.4） | Safari | ❌ | 长按图片 → 添加到照片 | 长按 / zip |
| Android | Chrome | ✅ | 系统面板 → 保存到相册 | 逐张 |
| Android | 微信内置浏览器 | ⚠️ 受限 | 多数情况会下载到本地 | zip |
| Mac | Safari | ✅ | 系统面板（局限） | zip |
| 桌面 | Chrome / Edge | ⚠️ 部分支持 | 下载 | zip |

> **微信内置浏览器**：Web Share API 在某些版本被禁用，会 fallback 到下载。后续可考虑加引导："建议用浏览器打开"。

## 7. 后续优化（不在 MVP 范围）

- **PWA**：添加到主屏幕，安装后窗口更原生
- **HEIC 服务端转码**：iPhone 直拍 HEIC 在 Android/桌面浏览器看不了
- **图片预加载策略**：viewer 进入时预热中等图，缓解大图首加载白屏
- **缩略图 srcset**：根据屏幕 DPR 选择不同分辨率（目前只一份 thumb）
- **微信浏览器引导**：检测到微信 UA 时给出"右上角 → 在浏览器打开"提示
- **iOS 全面屏 safe-area**：顶栏与底栏使用 `env(safe-area-inset-*)`
