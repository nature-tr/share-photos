# 在线共享相册 — 多端适配说明

> 本文档记录 Web / Android App / 微信小程序三端的设计差异与适配方案。

---

## 1. 三端对比

| 维度 | Web 前端 | Android App | 微信小程序 |
| --- | --- | --- | --- |
| 框架 | Vue 3 + Vite | Expo + React Native | Taro 4 + React |
| UI | TDesign Vue Next + UnoCSS | RN 原生组件 | SCSS + 内联 SVG |
| 图标 | TDesign Icon（UnoCSS 图标预设） | expo-image | 自建 SVG dataURL 库 |
| 状态管理 | Pinia | Zustand | Zustand |
| 路由 | Vue Router | expo-router | Taro 页面路由 |
| 登录持久化 | httpOnly Cookie | expo-secure-store | Taro StorageSync |
| 上传 | fetch + XHR | expo-image-picker + fetch | Taro.chooseMedia + Taro.uploadFile |
| 下载 | fetch → Blob → a.download | expo-media-library | Taro.downloadFile → saveImageToPhotosAlbum |
| 扫码 | — | expo-camera | Taro.scanCode |
| 二维码 | quickchart.io | quickchart.io | quickchart.io |

## 2. 小程序特化实现

### 2.1 图标方案

小程序无法使用 TDesign Icon（依赖 UnoCSS），自建了 `icons.ts` SVG dataURL 库：

- 24×24 viewBox，stroke 风格，与 TDesign Icon 视觉统一
- 每个图标函数支持动态着色 `iconAdd('#475569')`
- 共 13 个图标：add / list / logout / qrcode / scan / upload / folder / copy / link / arrowRight / imagePlus / download / trash / user

### 2.2 登录态持久化

小程序 zustand store 初始化时不保证 Taro API 就绪，采用：
1. `getUserFromStorage()` 直接读 storage（绕过 zustand 订阅）
2. 首页 `useState(() => getUserFromStorage())` 保底读取
3. `checkAuth()` 后台异步验证 + 自动刷新

### 2.3 全局进度追踪

- `task.store.ts`：zustand store 追踪所有活跃任务
- `GlobalProgress` 组件注册在 `App.tsx`，浮窗在任何页面可见
- 上传/下载完成后 3 秒自动消失

### 2.4 最近浏览校验

每次进入首页时后台并行校验所有历史分享：
- 已过期/结束/清除 → 自动从历史移除
- 被重命名 → 自动更新标题

### 2.5 滚动位置恢复

从最近浏览进入分享时带上 `fromHistory=1` 参数，详情页 `useLoad` 阶段读取上次滚动位置并恢复。其他入口（扫码/输入码）不触发。

## 3. 部署命令

### Web 前端

```bash
node scripts/build-frontend-release.mjs
bash scripts/deploy.sh frontend
```

### 后端

```bash
pnpm -F @photo/backend run build:release
bash scripts/deploy.sh backend
ssh root@www.dolmo.top 'pm2 reload photo-backend'
```

### 小程序

```bash
pnpm --filter @photo/miniapp build:weapp
# 微信开发者工具打开 miniapp/dist 目录预览/上传
```

### Android APK

```bash
cd app
pnpm -C ../shared build
npx expo prebuild --platform android --no-install
cd android && ./gradlew :app:assembleRelease --no-daemon --max-workers=2
bash scripts/upload-apk.sh
```
