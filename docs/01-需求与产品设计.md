# 在线共享相册 — 需求与产品设计文档

> 本文档用于沉淀项目背景、产品形态、功能边界与后续讨论锚点。
> 后续技术栈、代码结构等将在 `02-技术选型.md` 等独立文档中讨论。

---

## 1. 项目背景与动机

平时拍照后给朋友传图存在以下痛点：

- **微信传输**：自动压缩、画质损失严重，且批量发送速度慢。
- **网盘分享**：需要会员、操作繁琐、对方下载也不方便。
- **U盘 / AirDrop**：受限于物理在场或同生态系统。

因此希望自建一个 **基于 Web 的在线共享相册**，做到：

- 无需对方注册账号即可查看与下载。
- 不压缩画质（可下载原图）。
- 跨平台（浏览器即可访问）。
- 一次性 / 限时分享，自动清理，节省服务器资源。

---

## 2. 角色定义

| 角色 | 说明 |
| ---- | ---- |
| **分享者 (Sharer)** | 上传图片、设置有效期、获得分享码、管理自己的分享列表。 |
| **查看者 (Viewer)** | 通过分享码访问对应相册，预览缩略图、查看原图、下载（单张 / 批量）。 |

> 当前阶段：分享者是否需要登录账号待定（见第 6 节「待讨论问题」）。

## 3. 功能范围

### 3.1 基础功能（MVP，当前阶段目标）

#### 3.1.1 分享者侧

- **上传图片**
  - 支持多选、批量上传。
  - 支持常见图片格式（JPEG / PNG / HEIC / WebP 等，具体待定）。
  - 上传进度展示、断点/失败重试（可选，看实现复杂度）。
- **设置有效持续时长**
  - 例如：1 小时 / 6 小时 / 1 天 / 3 天 / 7 天等预设 + 自定义。
  - 到期自动失效。
- **生成分享码**
  - 短码（如 6~8 位）便于口头/文字传递。
  - 一份上传 = 一个分享码 = 一个相册集。
- **分享管理列表**（分享者本地可见）
  - 可看到自己创建过的所有分享。
  - 显示：分享码、创建时间、剩余有效时长、图片数量、状态（生效中 / 已过期 / 已手动结束）。
  - 操作：**一键续期**、**提前结束分享**。
- **过期清理**
  - 到期或手动结束后，服务器删除该分享对应的：
    - 所有原图
    - 所有预览图（缩略图）
    - 相关元数据（视实现而定，可保留极少日志）

#### 3.1.2 查看者侧

- **输入分享码访问**
  - 在首页/统一入口输入分享码 → 进入该相册预览页。
  - 失效/不存在的分享码 → 友好提示。
- **预览模式**
  - 默认仅加载 **缩略图 / 中等分辨率预览图**，避免直接加载原图导致流量过大。
  - 网格布局，支持懒加载 / 虚拟滚动（视图片数量决定）。
- **查看原图**
  - 点击单张缩略图 → 进入大图查看模式（支持左右切换）。
  - 在大图模式下可下载该张原图。
- **下载**
  - 单张原图下载。
  - **批量下载所有原图**（打包为 zip，或前端逐张拉取后打包，方案待定）。

### 3.2 已实现扩展功能（2026-06 完成）

- **账号体系**：邮箱注册/登录，JWT 双 token 认证。
- **跨设备分享管理**：云端账号，换设备也能看到所有分享。
- **贡献者系统**：他人可申请加入分享，创建者审核后协作上传。
- **分页加载**：大相册按需加载，默认 50 张/页，支持加载更多。
- **浏览历史**：自动记录查看过的分享，首页展示最近浏览。
- **滚动位置恢复**：重新进入分享时回到上次浏览位置。
- **补充图片**：分享者可为已有分享追加新图片。
- **二维码**：分享码一键生成二维码，保存或扫码访问。
- **限流保护**：2000 次/分钟，防止恶意请求。
- **WebP 优化**：缩略图生成 WebP 格式，体积减少 25-35%。
- **gzip/HTTP2**：API 响应压缩，多路复用加速。

### 3.3 扩展功能（未来迭代）

- **AI 筛图**：自动识别模糊、废片、重复图片。
- **点赞/评论**：社交互动功能。

## 4. 关键业务流程

### 4.1 分享流程

```
分享者
  └─ 选择图片
      └─ 选择有效期
          └─ 点击上传
              └─ 服务器：
                   1. 接收原图
                   2. 生成预览图（缩略图 / 中等分辨率）
                   3. 生成唯一分享码
                   4. 记录到期时间
              └─ 返回分享码给分享者
                   └─ 分享码进入「我的分享列表」
```

### 4.2 查看流程

```
查看者
  └─ 进入网页 → 输入分享码
      └─ 服务器校验：是否存在 / 是否过期
          └─ 有效：返回该相册的预览图列表（不返回原图）
              └─ 查看者浏览缩略图
                   ├─ 点开 → 加载原图（按需）
                   ├─ 单张下载
                   └─ 批量下载（打包）
          └─ 无效：提示「分享码不存在或已过期」
```

### 4.3 生命周期管理

```
[创建] → [生效中] ──到期──→ [已过期] → [清理任务删除资源]
              │
              ├──分享者续期──→ 延长到期时间，仍处于「生效中」
              └──分享者主动结束──→ [已手动结束] → [清理任务删除资源]
```

## 5. 非功能性需求（初步设想，待讨论）

| 维度 | 初步想法 |
| ---- | -------- |
| **画质** | 原图无损存储；预览图允许有损压缩。 |
| **存储** | 到期立即清理，避免长期占用。 |
| **性能** | 预览图懒加载；批量下载需考虑大文件场景。 |
| **安全** | 分享码具备一定随机性，难以爆破；可考虑访问频率限制。 |
| **隐私** | 不做公开搜索；仅凭分享码访问；过期即删。 |
| **成本** | 服务器资源（尤其是带宽 / 存储）尽量可控。 |
| **跨平台** | 纯浏览器访问，移动端 / 桌面端体验都需良好。 |

## 6. 当前需要讨论 / 决策的问题

> 这些问题不在此文档中给出结论，将在后续的技术栈与架构讨论中逐步明确。

1. **分享者是否需要账号体系？**
   - 方案 A：无账号，分享列表通过浏览器本地（localStorage / 本地数据库）记忆。
   - 方案 B：轻量账号（邮箱 / 第三方登录），跨设备查看分享列表。
2. **部署形态？**
   - 自部署（自有服务器 / VPS） vs 云服务（对象存储 + Serverless 等）。
3. **存储方案？**
   - 本地磁盘 vs 对象存储（S3 / OSS / COS / MinIO 等）。
4. **预览图生成时机？**
   - 上传时同步生成 vs 异步队列生成。
5. **批量下载实现？**
   - 服务端打包 zip vs 前端 JSZip 流式打包。
6. **技术栈选型？**
   - 前端框架、后端语言/框架、数据库、对象存储、构建/部署方案。
7. **分享码格式与防爆破策略？**
   - 长度、字符集、是否加访问令牌、是否加访问频率限制。
8. **HEIC 等特殊格式是否需要服务端转码？**
