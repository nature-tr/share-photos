## 1. 最终技术栈

### 1.4 后端

| 项 | 选型 | 备注 |
| --- | --- | --- |
| 运行时 | **Node.js 20 LTS** | |
| 语言 | **TypeScript** | |

### 1.5 工程

| 项 | 选型 | 备注 |
| --- | --- | --- |
| 包管理 | **pnpm 9** | |

### 1.6 文件存储

- **本地文件系统**，目录约定：

```
storage/
  originals/<shareId>/<photoId>.<ext>     # 原图
  previews/<shareId>/<photoId>.jpg        # 缩略图：短边 ~400, JPEG q70
  previews/<shareId>/<photoId>.webp       # 缩略图 WebP 版
  mediums/<shareId>/<photoId>.jpg         # 中等预览图：长边 ~1600, JPEG q80
  mediums/<shareId>/<photoId>.webp        # 中等图 WebP 版
```

## 4. 默认约定

- **Node 版本**：≥ 20 LTS
- **包管理**：pnpm 9
- **TypeScript**：`strict: true`
- **接口风格**：RESTful + JSON
- **响应格式**：成功 `{ data: <T> }`，失败 `{ error: { code, message, details? } }`
- **时间格式**：毫秒级 UNIX 时间戳
- **ID 生成**：nanoid 16 字符；分享码 8 位（去 0/O/1/l/I）

## 5. 实施备注

### 5.1 better-sqlite3 版本锁定

锁定到 **11.5.0**，覆盖 Node 20/22/24 的 prebuild。

### 5.2 sharp 0.33 兼容

- `limitInputPixels` 链式方法已移除 → 改用 metadata 后手动检查像素
- `concurrency` 静态方法已废弃 → 移除调用

### 5.3 数据库迁移

`drizzle-kit generate` 生成迁移 → 后端启动时自动 `runMigrations()`。
